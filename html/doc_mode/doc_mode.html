  • Compatibility View is the way IE try to implement backwards Compatibility when newer IE version is transferring itself to W3C spec
  • Compatibility View can't set by pages, it is controlled by the client or IT policy
  • in Compatibility View (old IE) document.getElementById is case insensitive but modern browser and new IE, it is case sensitive.
  • Compatibility View is browser mode, not document mode. document mode is controlled by pages.
  • child window which is opened by window.open can have different browser and document mode from base window
  • parent doc and child doc in frames can have different doc mode, but unfortunately in IE console you can only see parent doc mode
  • doc mode will affect javascript more than Compatibility View, if doc mode is highe enough, most javascript code will work
